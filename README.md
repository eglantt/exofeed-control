# ExoFeed Control

Приложение для учёта кормления и линьки животных в террариумном хозяйстве. Работает полностью в браузере, без сервера и без интернета (кроме самой первой загрузки страницы) — вся база хранится локально в браузере устройства (`localStorage`).

Текущая версия: **v1.5.5-alpha**.

## Зачем размещать на GitHub Pages

Если открывать `index.html` напрямую с телефона (двойным тапом / через «Файлы»), некоторые современные версии Android/iOS блокируют локальное хранилище (`localStorage`) для файлов, открытых с диска (`file://`) — из-за этого приложение может не запускаться.

При размещении через GitHub Pages страница открывается по обычному адресу `https://...github.io/...`, для которого это ограничение не действует — работает как любой обычный сайт.

**Важно:** сама база животных при этом никуда на GitHub не отправляется и не хранится там — она остаётся только в браузере того устройства, с которого вы её вносите. Открыв сайт с двух разных устройств (телефон и планшет, например), вы получите две независимые базы — для переноса данных между ними по-прежнему нужен экспорт/импорт (кнопки «💾 Бэкап» и «📂 Импорт» внутри приложения).

## Как разместить (один раз, 5 минут)

1. Создайте новый репозиторий на [github.com](https://github.com/new) — например, `exofeed-control`. Можно публичный или приватный (для приватного репозитория GitHub Pages доступен только на платных тарифах — если тариф бесплатный, выбирайте публичный репозиторий; на видимость самой базы животных это не влияет, см. выше).
2. Загрузите **все файлы из этого комплекта** в корень репозитория, сохранив структуру папок (кнопка **Add file → Upload files** на странице репозитория — можно перетащить сразу всё, включая папку `icons`, либо через `git push`, если пользуетесь консолью):
   - `index.html`
   - `manifest.json`
   - `sw.js`
   - `icons/icon-192.png`
   - `icons/icon-512.png`
   - `icons/icon-512-maskable.png`
   - `icons/apple-touch-icon.png`
3. Зайдите в **Settings → Pages** этого репозитория.
4. В разделе **Build and deployment** выберите **Source: Deploy from a branch**, ветку **main**, папку **/ (root)** → **Save**.
5. Через 1–2 минуты GitHub покажет ссылку вида `https://<ваш-логин>.github.io/exofeed-control/` — это и есть рабочий адрес приложения.

## Установка как приложение (PWA) на телефон

После размещения на GitHub Pages сайт можно установить на главный экран как обычное приложение — со своей иконкой, без адресной строки браузера, и с работой без интернета после первого открытия.

**Android (Chrome):** откройте ссылку → в меню браузера (⋮) выберите **«Установить приложение»** (или Chrome сам покажет баннер с таким предложением).

**iPhone/iPad (Safari):** откройте ссылку → нажмите кнопку «Поделиться» (квадрат со стрелкой вверх) → **«На экран «Домой»»**.

После установки на экране появится иконка ExoFeed — открывается как отдельное приложение. Данные (база животных) при этом точно такие же, как в браузере — это не отдельное хранилище, а тот же `localStorage` того же сайта.

## Обновление версии в будущем

Когда появится новая версия — замените файл `index.html` (и, если менялись, `sw.js`/`manifest.json`/иконки) в репозитории тем же способом (Add file → Upload files, с подтверждением перезаписи) или через `git push`. Ссылка на сайт останется той же, иконка на телефоне — тоже. Формат хранения базы совместим между версиями — при первом открытии новой версии старые данные подхватятся автоматически (миграция встроена в само приложение).

## Локальная разработка (не обязательно)

Если захотите вносить правки в код самостоятельно — файл `index.html` полностью автономен (весь HTML/CSS/JS в одном файле), можно редактировать любым текстовым редактором и открывать в браузере с компьютера напрямую (на десктопе `file://`-ограничение на localStorage обычно не действует, в отличие от мобильных браузеров).
